# Docs

Project documentation for Klean Reader.

## Purpose

This folder captures architecture, decisions, setup, API contracts, testing approach, release notes, and UX artifacts.

## How To Use

- Prefer short, practical notes over long prose.
- Add date and context when recording major decisions.
- Link related docs to keep context connected.

## Structure

- architecture/: system and app architecture notes
- adr/: architecture decision records
- api/: API contracts and integration notes
- setup/: local development setup and environment notes
- testing/: testing strategy and quality gates
- release/: deployment and release workflows
- product/: feature notes, user flows, and acceptance criteria
- ux/: wireframes, diagrams, and interaction notes
